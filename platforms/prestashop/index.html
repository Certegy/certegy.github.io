Setting up Oxipay on a PrestaShop Store

This document outlines the steps needed to install Oxipay on your PrestaShop store. Oxipay enables customers to purchase items using their credit cards on an interest free payment plan.

The Oxipay plugin has been developed and tested against PrestaShop version 1.6.1.9 Please note however, that it should also compatible with all 1.6.X and 1.7.X versions.

Installing Oxipay

You will need your Merchant ID and an Encryption Key handy before continuing with the installation procdure.

The plugin can be installed automatically via PrestaShop's admin panel. This section details how to upload the plugin .zip file, install and then configure its various parameters for it to work properly.

Installation of the plugin requires you to be able to access the store's admin area. If you are having trouble in access your PrestaShop's admin area, then please consult PrestaShop's official support documentation or your IT department to resolve the issue.

Installation Procedure

1 - Download a copy of the Oxipay plugin for PrestaShop. To download your copy, navigate to the following URL, click on the Clone or download link in the top right then click on Download ZIP from the menu that appears. This will download the plugin as a ZIP file into your default installation directory.

https://github.com/oxipay/oxipay-prestashop

2 - Log into your PrestaShop's admin area using an Email address and a Password combination that has already been configured as an admin.

3 - Once logged in, you will be presented with your store's dashboard. From the side menu that appears to the left, hover over Modules and Services and then click on Modules and Services from the sub-menu that appears.

4 - This wll take you to the Modules and Services page and wil display a list of modules that are available and can be installed/configured on your store.

5 - Scroll downwards and bring the Modules List panel into focus and ensure that both filtering drop-downs are configured as follow - i.e. they are set to display Installed & Not Installed plugins as well as those that are Enabled & Disabled.

You might be presented with a warning at the top of the page indicating that you don't have Bank wire or Payment by check enabled on your site. If you don't intend on processing payment using either of those methods, then you can ignore this warning message

6 - Scroll to the top of the page and then click on the Add a new module icon. This will open up the Add a new module panel.

7 - From within the Add a new module panel, click on Choose a file. This will allow you to specify the folder or directory that contains the Oxipay .zip plugin. Once selected, click on Upload this module to initiate the uploading of the plugin.

8 - If the installation of the plugin was successful, then you will be presented with a success alert at the top of the page indicated that the module was successfully donwloaded.

9 - To confirm that thte plugin was successfully downloaded, scroll downwards until the Modules List panel is in focus. Then type in oxipay in the search field at the top left corner of the panel as per the screenshot below. This should filter out all other modules and only leave the Oxipay Prestashop module in focus.

Depending on the version of the plugin that you have downloaded, the version number displayed on the Oxipay prestashop plugin might be different to the one in the screenshot. This document has been prepared using version 1.1.0.

10 - Click n the green Install button next to the Oxipay prestashop listing.

11 - Clicking on install will display a prompt requesting that you confirm that you want to go ahead with the installation of the plugin even though it has not been verified by PrestaShop. Click on the orange Proceed with the Installation button.

You can ignore the error message that indicates that the Oxipay module could not be verified. This error is displayed because the module was installed locally from your machine, rather than through PrestaShop's module marketplace.

12 - If the installation is successful, you will be re-directed to the the Oxipay module configuration page with a message at the top of the page indicating that the module was installed successfully.

Configuring Oxipay

This section assumes that you have just finished installing the Oxipay plugin and are still on the Oxipay Configuration page that you were automically re-directed to after installation.

This section details the various parameters that you would need to configure including this such as the plugin's logo, Oxipay endpoints to communicate with as well as the whether to enable Test Mode or not.

  • Title This configures the plugin title that gets displayed to the end user during checkout. By default it is configured to Oxipay but you have the option of changing it to something else such as Oxipay Australia or Oxipay Payment Gateway for instance.

  • Logo This controls the Oxipay Logo that gets displayed

  • Description This configures the plugin description text that gets displayed to the end user as part of the checkout process. By default, it is configured to be Breathe easy with Oxipay, an interest-free installment payment plan. but you do have the option of changing it to another text string.

  • Oxipay Gateway URL This specifies the Oxipay end-point the plugin would need to communicate with for it to be able to process payments via Oxipay. Note that this is the live Oxipay Gateway URL meaning transactions processed over it are real transactions that end up in real dollar amounts being deducted.

  • Oxipay Sandbox Gateway URL This specifies the test Oxipay end-point that the plugin would need to communicate with when Test Mode is enabled. This is mainly used for troubleshooting purposes particularly if we want to simulate transactions on your PrestaShop store without having real dollar amounts debited from Credit Cards.

  • Merchant ID This is the Oxipay Merchant ID that you are provided with as part of your on-boarding when signing-up with Oxipay.

  • API Key The API Key or the Encryption key is a case-sentitive alpha-numeric that is used to authenticate payment requests originating from your store. It is similar to passwords that you use to login into your online email account for instnace, and so it is vital that you protect it and don't provide it to un-authorised individuals. We might also re-cycle your API and provide you with a new one if un-authorised individuals get hold of it or if it has been compromised.

  • Test Indicates whether to process payments over the live Oxipay Gateway URL resulting in real transaction; or to process it over the Oxipay Sandbox Gateway URL which only produces simulated (test) transaction with no real dollar amounts being debited. By default it should be set to No unless you are attempting to diagnose issues with our help.

Once you have finished making the necessary configuration changes or updates, click on the Save button on the bottom left. If successful you will be presented with a success message at the top of the page, similar to the screenshot below.

Quick Config Actions

PrestaShop allows you to perform quick actions on your installed modules includes things such as disabling the plugin on specific devices or uninstalling it. To perform one of the quick actions:

1 - Log into your PrestaShop admin panel.

2 - However over the Modules and Services option on the left side menu and then click on Modules and Services item on the sub-menu that appears.

3 - This will re-direct you to the List of Modules page.

4 - Scroll downards until you bring the Modules List panel into focus and then type Oixpay in the search field in the panel's top left corner. This will bring the Oxipay module into focus and filter out all the other modules.

5 - Click on the drop-down arrow next to the Configure button. This will open up a drop down menu similar to the one show in the screenshot underneath.

Below is a quick summary of the various options available to you

  • Disable : Disables the plugin but leaves it installed on your PrestaShop. You can re-enable it with its existing settings at a later time. Oxipay, however, will not appear as a payment option on your store once disabled.
  • Disable on mobiles : This option disables Oxipay only on mobile devices (smartphones such as iPhones for instance). This does not apply to tablets such as iPads or desktop computers.
  • Diable on tables : Similar to above but disables Oxipay on tablets and not smartphones or desktop computers.
  • Disable on computers : Disables Oxipay on desktop computers only.
  • Reset : Resets the Oxipay module to its original state without any of the configuration settings that have been applied post installation.
  • Uninstall : Uninstalls the plugin from your store. However, the plugin will still be available and you will be able to install it at a later time if you chose to without having to re-upload it again.
  • Mark as Favorite : Marks Oxipay as one of your favourite plugins so that it becomes available from within the Favourites category list on the left hand side of the Modules List panel.

  • Delete : Compeletely removes the plugin from your store. The plugin will no longer be available for installation again and the only way to add Oxipay back to your store as a payment method would be to perform the instructions in the Installing Oxipay section of the document again.

Upgrading Oxipay

To upgrade your Oxipay PrestaShop plugin and then install the newer version. To uninstall the plugin you would need to:

1 - Open up the Quick Config Actions using the instructions in the Quick Config Actions section of this document.

2 - From the drop-down menu that appears, click on Delete.

3 - This will display a pop-up in your browser similar to the one show in the image underneath.

4 - Click on OK. This will initiate the deletion of the Oxipay plugin. Once completed, you will be presented with a success message at the top of the page similar to the one show in the screenshot below.

5 - You can now download the newer version of the Oxipay PrestaShop plugin and install it as per the instructions in the Installing Oxipay section of this document.
