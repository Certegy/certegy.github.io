API Reference

Communicating with Oxipay in order to process a transaction via our payment gateway involves POSTing a request to the Oxipay endpoint. This can be viewed as an authorisation request that is performed by the shopping cat in order to process the payment via Oxipay.

There are two Oxipay endpoints that transaction information can be posted to and these are the live (production) payment gateway and the test (sandbox) payment gateway.

Oxipay Endpoints

The Production Oxipay endpoint is :
https://secure.oxipay.com.au/Checkout?platform=Default

The Sandbox Oxipay endpoint is :
https://xpozsecure.certegyezipay.com.au/Checkout?platform=Default

Request POST Format

Posting to the Oxipay endpoint, regardless of whether this is done in the live or test environments, should be done in the application/x-www-form-urlencoded format whereby the query string will represent the body of the HTTP message that is sent to Oxipay. The various key-value pairs that represent the transaction payload are separated by the & character whilst the key-value pairs are separated by the = character. Note that Oxipay adopts the convention of prefixing the various key names with x_ and we anticipate that you will adopt a similar convention when integrating Oxipay into your shopping cart.

Below is an overview of the various key-value combination that can be passed to Oxipay (Request Values), a description of what they are as well as an indication of whether they are mandatory or optional.

Sample POST

Below is a sample request that might be posted to an Oxipay endpoint. Note that the POST is in the application/x-www-form-urlencoded format.

x_reference=123&x_account_id=1&x_amount=100.00&x_currency=AUD
&x_url_callback=sample_callback_url&x_url_complete=sample_complete_url
&x_shop_country=AU&x_shop_name=Sample+Shop&x_test=true
&x_customer_first_name=first&x_customer_last_name=last
&x_customer_email=sample%40email.com&x_customer_billing_country=AU
&x_customer_billing_city=Adelaide&x_customer_billing_address1=97+Pirie
&x_customer_billing_address2=St&x_customer_billing_state=SA
&x_customer_billing_zip=5000&x_invoice=%123
&x_description=Sample+Store+-+%123
&x_url_cancel=sample_cancel_url&x_signature=dummy_signature

Response POST

Once a transaction has been processed by Oxipay, Oxipay will communicate with the shopping platform using the x_url_callback URL that is specified as part of the request POST. The response is also sent in the application/x-www-form-urlencoded format as an asynchronous POST. Similar to the request POST, the response POST includes key-values pairs that are specific to that transaction and indicate things such as the outcome of that particular transaction whether it has failed or is pending another action for instance.

Response POST key-value pairs

Below is an overview of the various response key-value pairs that Oxipay returns after it has finished processing a transaction. Note that some of these key-value pairs echo corresponding key-value pairs in the request that Oxipay receives - as is the case with x_currency for instance.

x_amount Required Represents the transaction's total amount including any taxes and shipping costs decimal 99.90
